# MarkdownText Component Documentation

The `MarkdownText` component is a robust, custom-built Markdown parser designed to render the text generated by the Anthropic AI model directly into native React Native elements.

## 1. 🧠 Logic / Business Logic

### Custom Parser Engine
- **`parseMarkdown`**: Manually tokenizer iterating line-by-line across the provided `content` string.
  - Recognizes Headings (`#` to `######`), mapping them to native `Text` tags with varying explicit font sizes.
  - Understands `>` Blockquotes, rendering them with a left-colored border.
  - Supports List parsing (Bullets `-`/`*` and Numbered `1.`).
  - Capable of identifying Task lists (`[ ]` vs `[x]`) and rendering Unicode checkboxes.
  - Extracts Multi-line code blocks bounded by ` ``` ` and presents them with mono-spaced fonts and contrasting backgrounds.

### Inline Styles
- **`parseInlineMarkdown`**: A regex-driven helper that parses string fragments.
  - Isolates and styles text for **Bold**, *Italic*, ~~Strikethrough~~, and `inlineCode`.
  - Ensures fragments don't overlap, dynamically injecting styled `<Text>` wedges inside the parent `<View>`.

### Selectable Mode
- Supports a `selectable` boolean prop. When `true`, it strips all styling and markdown decorators mathematically, piping the raw core string into a disabled `TextInput` component. This bypasses React Native clipping restrictions and gives users access to OS-level long-press highlighting and copy menus.

## 2. 🔌 API Integration

*This component receives direct text payloads resolved from `AnthropicService` (Claude API) via its parent containers (like `ai.tsx`).*

---

## 3. 🎨 UI / User Interface

### Aesthetic Design
- Adheres entirely to a dark theme. Text defaults to absolute `#FFFFFF`.
- **Code Blocks**: Uses `rgba(255, 255, 255, 0.1)` backgrounds and bright green `#00DC82` mono fonts (`Courier`) to instantly distinct syntax from prose.
- **Lists**: Mathematically calculates indentation based on space depth in the raw markdown, cascading the margin seamlessly.
